传统单体架构：SSH，SSM，前后端不分离，前端使用 jsp、thymeleaf，部署在 tomcat 的项目。

分布式架构：单体架构的演变，将传统项目根据业务进行拆分，比如用户系统，短信系统，支付系统，订单系统，降低项目的耦合度。

SOA 面向服务架构：分布式架构的演变，将相同的业务逻辑组装成一个公共的服务，解决冗余性问题，面向服务开发(面向接口开发)。分布式架构一个项目可能会链接多个数据库，有多个数据源，从而查询不同服务数据信息，SOA 架构则是通过接口调用查询其他服务的信息，服务与服务之间采用 RPC 远程调用。

缺点是服务化的治理不够完善。

微服务架构：SOA 架构的演变，对服务的拆分更加精细，服务与服务之间互不影响，每个服务有单独的数据库，独立部署。核心是注册中心，使服务治理更加高效，加入网关，配置中心，使服务的鉴权，保护更加简单，服务与服务之间的通讯采用 RESTful 方式数据传输协议采用 http + json。

前后端分离架构，专业的人去做专业的事情，提高开发的效率。
